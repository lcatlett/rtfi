# Product Brief: LLM Instruction Compliance Observability

**Author:** Generated with Claude (requires validation)
**Date:** 2026-01-31
**Status:** Draft for market demand assessment

---

## Problem Statement

Large Language Models in production workflows ignore explicit instructions at unpredictable rates. Users write system prompts, CLAUDE.md files, custom instructions, and behavioral guidelines - and the AI disregards them without notification.

**The failure mode:**
1. User provides explicit instruction (e.g., "Always clarify ambiguous requests before proceeding")
2. AI acknowledges understanding of the instruction
3. AI violates the instruction during execution
4. User discovers violation only through downstream failure (bad outputs, wasted work, incorrect results)
5. No system flagged the non-compliance

**Current state:** Users have no visibility into whether their instructions were followed. Compliance is discovered retroactively through output quality problems, not proactively through monitoring.

---

## Evidence of Problem Severity

### Academic Research Confirms Widespread Non-Compliance

- **[MOSAIC Benchmark](https://arxiv.org/html/2601.18554)**: Modular assessment reveals "model-specific positional biases, such as primacy and recency effects" - instructions at different positions are followed at different rates
- **[The Instruction Gap](https://www.arxiv.org/pdf/2601.03269)**: "Violation counts ranging from 660 to 1330" across leading LLMs in enterprise scenarios; "models that excel in general benchmarks may struggle with specific enterprise instruction requirements"
- **[InFoBench](https://aclanthology.org/2024.findings-acl.772/)**: "Even large-scale instruction-tuned LLMs fail to follow simple instructions in zero-shot settings"

### Real-World Impact

This isn't theoretical. In the session that prompted this brief:
- 42 research documents were generated by Claude
- Claude was explicitly instructed to "clarify before acting on ambiguous requests"
- Claude ignored the instruction and spawned 11 parallel agents without clarification
- All 42 documents were unusable (grounded in web search instead of institutional knowledge)
- Discovery occurred only after significant time investment reviewing outputs

## The Unsolved Problem

**No tool answers the question: "Did the AI follow the instructions I gave it in this session?"**

Specifically:
1. **No instruction extraction** - No system parses user instructions (system prompts, CLAUDE.md, custom guidelines) into verifiable rules
2. **No behavioral mapping** - No system maps AI actions to instruction requirements
3. **No compliance scoring** - No system measures per-instruction compliance rates
4. **No real-time alerting** - No system notifies when instructions are violated during execution
5. **No compliance analytics** - No system tracks which instructions are ignored most often, under what conditions

---

## Why Hasn't This Been Solved?

### 1. The Problem Is Hard to Define

"Instruction following" is fuzzy. Unlike hallucination detection (compare output to source) or safety filtering (match against blocklists), instruction compliance requires:
- Parsing natural language instructions into testable conditions
- Mapping AI behavior to those conditions
- Handling ambiguity (was this a violation or a legitimate interpretation?)

### 2. Tooling Ecosystem Assumes Compliance

Claude Code's documentation explicitly states CLAUDE.md is "advisory, not deterministic." The ecosystem treats instruction non-compliance as expected behavior, not a bug to monitor.

### 3. Focus on Output Quality, Not Process Compliance

The market prioritizes: "Is the output good?" not "Did the AI follow the process to produce it?"

For many applications, this is fine. For enterprise workflows where *how* the work is done matters (audit trails, methodology compliance, safety protocols), it's insufficient.

### 4. No Standard Instruction Format

Instructions are unstructured natural language. Without a schema, automated compliance checking requires semantic understanding - which means using another LLM, which introduces its own compliance uncertainty.

### 5. LLM Providers Don't Ship This

Anthropic, OpenAI, and others don't provide instruction compliance metrics. The closest is structured outputs (schema enforcement), but that constrains *format*, not *behavior*.

## Proposed Solution: Compliance Risk Score

### Core Insight

You can't make the AI follow instructions. But you can **predict when it's likely to fail** and insert human checkpoints at the right moments.

This shifts the problem from "fix AI behavior" (impossible) to "manage AI risk" (tractable).

### Research-Backed Risk Factors

The [MOSAIC benchmark](https://arxiv.org/html/2601.18554) identified factors that affect instruction compliance:

| Factor | Effect on Compliance |
|--------|---------------------|
| **Number of instructions** | More instructions → lower compliance per instruction |
| **Instruction position** | Primacy and recency effects - middle instructions ignored more often |
| **Constraint interactions** | Some instruction combinations are synergistic, others conflicting |
| **Context length** | Longer context → earlier instructions deprioritized |

### Proposed Risk Model

Track these factors in real-time during AI sessions:

```
Compliance Risk = f(
    context_length,      # Tokens in context when decision made
    instruction_count,   # Number of active rules
    autonomy_depth,      # Steps between human input and action
    agent_fanout,        # Number of parallel agents spawned
    decision_points      # Number of tool calls executed
)
```

Higher scores = higher probability of instruction non-compliance = require more verification.

### How It Would Work

1. **Real-time tracking** - Monitor risk factors as session progresses
2. **Risk score calculation** - Update score with each action
3. **Threshold alerts** - When score crosses configurable threshold:
   - Alert: "This session is in high-risk territory for instruction non-compliance"
   - Checkpoint: "Verify outputs before continuing"
   - Confirmation: "11 agents about to spawn. Confirm methodology is correct?"
4. **Post-session audit** - Log risk trajectory for compliance reporting

### Applied to the Failure Case

For the research failure that prompted this brief, this system would have flagged:

| Factor | Value | Risk Contribution |
|--------|-------|-------------------|
| Agent fan-out | 11 parallel agents | High |
| Context load | Large (PRD sections + research queries) | High |
| Autonomy depth | Multiple steps without confirmation | High |
| Decision points | Many tool calls before verification | High |

**Result:** Risk threshold exceeded before agents spawned. System would have paused and required: "Confirm methodology is correct before spawning 11 agents."

### Key Differentiation

| Existing Tools | This Approach |
|----------------|---------------|
| Monitor outputs after the fact | Predict risk before failure |
| Detect hallucinations in content | Detect conditions that cause non-compliance |
| Retroactive quality assessment | Proactive intervention points |
| "Was the output good?" | "Is the session at risk of ignoring instructions?" |

## Solution Components

### 1. Risk Tracker (Core)
- Hook into AI execution environment (Claude Code hooks, API middleware, etc.)
- Track risk factors in real-time
- Calculate rolling compliance risk score
- Fire alerts at configurable thresholds

### 2. Instruction Registry (Optional Enhancement)
- Parse user instructions into testable conditions
- Weight instructions by criticality
- Enable per-instruction compliance tracking

### 3. Behavior Logger
- Log all tool calls, outputs, decisions with context
- Enable post-session compliance audit
- Generate audit trails for regulatory compliance

### 4. Analytics Dashboard
- Track risk scores over time
- Identify patterns (which workflows are riskiest?)
- Correlate risk with actual compliance failures
- Refine risk model based on observed data

### 5. Integration Layer
- Claude Code hooks
- API middleware for other LLM providers
- IDE plugins (Cursor, VS Code)
- CI/CD integration for automated workflows

## Summary

**The problem:** LLMs ignore instructions without notification. Users discover failures retroactively.

**Why unsolved:** Hard to define, ecosystem assumes compliance, focus on outputs over process.

**The insight:** You can't make AI follow instructions. But you can predict when it's likely to fail based on measurable risk factors (context length, agent fan-out, autonomy depth, instruction count).

**The product:** Real-time compliance risk scoring that flags high-risk sessions before failures occur, enabling proactive human checkpoints instead of retroactive discovery.

- Regulatory pressure (EU AI Act, California SB 243/AB 489)
